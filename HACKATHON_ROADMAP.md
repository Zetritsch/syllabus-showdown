# Syllabus Showdown — Hackathon Roadmap

## Product promise

Turn any study material into an adaptive multiplayer game show with interactions that go beyond multiple choice.

## Complete

- Validated, source-grounded Game Pack schema
- Text, Markdown, selectable PDF, scanned PDF, and diagram input
- Cost-controlled GPT generation with visible usage estimates
- Sequence Rush, Sort Reactor, Hotspot Hunt, Model Assembly, Connection Clash, and Confidence Battle
- Adaptive body, cell, cycle, and system model templates
- Deterministic judge demo that does not require API credit
- Supabase Realtime lobby, host progression, player scores, late join, and reconnect recovery
- QR room invites
- Persistent German and English interface and generation language
- Source evidence shown after every answer
- Per-round visual identity, game-show feedback, sound toggle, and final podium

## Submission-critical next

1. Run a complete two-device production QA pass and fix every blocker.
2. Add curated sample PDFs so judges can see anatomy and diagram generation quickly.
3. Capture final 3:2 screenshots for the project gallery.
4. Record a 90–120 second demo video with a tight judge narrative.
5. Finish the project story, architecture summary, Built With tags, repository link, and live link.
6. Add a concise privacy/cost notice and sharpen empty/error states.

## Stretch after submission essentials

- More generated model templates beyond body, cell, cycle, and system
- Teacher recap with misconception clusters and learning recommendations
- Team mode and asynchronous challenge links
- Saved Game Pack library and replay history

## Judge demo sequence

1. Show the source PDF and choose the output language.
2. Generate a validated pack and point out the displayed API cost.
3. Open a room and let a phone join through the QR code.
4. Play Sort Reactor, Hotspot Hunt, and Anatomy Assembly.
5. Answer a misconception with high confidence to trigger adaptive remediation.
6. End on the live podium and source-grounded learning recap.
